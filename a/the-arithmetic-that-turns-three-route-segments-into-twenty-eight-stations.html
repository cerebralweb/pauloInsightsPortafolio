THE ARITHMETIC THAT TURNS THREE ROUTE SEGMENTS INTO TWENTY EIGHT STATIONS

In this article ❧ Route and coverage planning rarely starts with one clean measurement. Operations teams combine segments recorded in different ways, then turn the total into a count of equipment or staff placed at a fixed interval. This problem shows how adding unlike fractions and dividing by a rate models a real placement decision, and how Python turns that logic into a reusable tool.

The Problem

A logistics team is mapping a delivery route that runs through three zones. The first zone covers 2.5 km, the second covers 2 and 1/3 km, and the third covers 2 and 1/6 km. The team must place a scanning station every quarter kilometer along the full route.

How many scanning stations are needed?

  • a) 28
  • b) 5
  • c) 4
  • d) 2
Click to Reveal the Answer

Correct Answer: a) 28 ✅

Math Solution

Add the three segment lengths on a common base to reach the full route of 7 km, then divide by the station spacing. Dividing by a quarter is the same as multiplying by four, so the total maps straight to the station count.

Dividing by 0.25 is the same as multiplying by 4.

Step-by-Step Breakdown

The following sequence shows how the final amount is derived:

  1. Whole kilometers: 2 + 2 + 2 = 6
  2. Fraction kilometers: 1/2 + 1/3 + 1/6 = 1
  3. Total route: 6 + 1 = 7 km
  4. Station count: 7 ÷ 0.25 = 7 × 4 = 28

Three Solution Routes

The same answer can be reached three ways. Each one combines the lengths to 7, then turns 7 into 28.

  1. Mental math. Each zone is at least 2 km, so 2 × 3 = 6 km, which already covers 24 stations. The leftover fractions 1/3 + 1/6 = 3/6 = 1/2, plus the remaining 1/2, give 1 more km and 4 more stations. 24 + 4 = 28.
  2. Decimals. Add the whole numbers, then the fractions. 2.5 + 2.333 + 2.166 = 7, then 7 × 4 = 28. A third has no clean decimal, so keep the full repeating value or the total falls just short of 7.
  3. Direct. Total 7 km, and dividing by a quarter is multiplying by 4, so 7 × 4 = 28.

Historical Background

Combining parts measured in halves, thirds, and sixths is one of the oldest recorded calculations. Egyptian scribes wrote almost every fraction as a sum of unit fractions, each with one on top, and the Rhind Mathematical Papyrus copied around 1550 BCE is full of such sums. The identity 1/2 + 1/3 + 1/6 = 1 is exactly the kind of unit fraction sum it drills, so the zone fractions here reproduce a piece of arithmetic more than three thousand years old. The same instinct survives in commerce and engineering, where a quantity is split into convenient parts, recombined into one total, then divided by a fixed rate to plan supplies, staff, or equipment along a line.

Python Version

Two scripts solve the same problem. The first uses plain arithmetic. The second keeps the fractions exact so thirds and sixths never drift, which makes it reliable for any route and any interval.

Option 1, pure Python.

# What this does. Counts the scanning stations a route needs.
# Where it came from. A logistics team maps a route through three zones and places a station every quarter km.
# The math idea. Add the three lengths, then divide by the spacing.
# You type in. The three zone lengths and the spacing, all in km.
# It prints. How many scanning stations the route needs.

# This function does the math.
def stations_needed(zone1, zone2, zone3, spacing):
    # Add the three zone lengths to get the full route.
    total = zone1 + zone2 + zone3
    # Divide the route by the spacing to get the station count.
    return total / spacing

# Tell the reader what we will work out.
print("Let's count the scanning stations the route needs.")

# Ask for each zone length in km.
zone1 = float(input("Zone 1 length in km "))
zone2 = float(input("Zone 2 length in km "))
zone3 = float(input("Zone 3 length in km "))

# Ask for the spacing between stations in km.
spacing = float(input("Spacing between stations in km "))

# Do the math.
result = stations_needed(zone1, zone2, zone3, spacing)

# Show a whole number when the result has no leftover.
result = int(result) if result.is_integer() else result

# Print the answer with words that fit the scenario.
print("Scanning stations the route needs", result)

Option 2, exact fractions with the Fraction class.

# What this does. Counts the scanning stations a route needs, with exact fractions.
# Where it came from. A logistics team maps a route through three zones and places a station every quarter km.
# The math idea. Add the three lengths as exact fractions, then divide by the spacing.
# You type in. The three zone lengths and the spacing, as a fraction like 7/3 or a decimal like 2.5.
# It prints. How many scanning stations the route needs.

# Bring in a tool that keeps fractions exact, so a third stays a third.
from fractions import Fraction

# This function does the math.
def stations_needed(zone1, zone2, zone3, spacing):
    # Add the three zone lengths to get the full route.
    total = zone1 + zone2 + zone3
    # Divide the route by the spacing to get the station count.
    return total / spacing

# Tell the reader what we will work out.
print("Let's count the scanning stations the route needs.")

# Ask for each zone length. A fraction like 7/3 or a decimal like 2.5 both work.
zone1 = Fraction(input("Zone 1 length in km "))
zone2 = Fraction(input("Zone 2 length in km "))
zone3 = Fraction(input("Zone 3 length in km "))

# Ask for the spacing between stations.
spacing = Fraction(input("Spacing between stations in km "))

# Do the math. The result stays exact.
result = stations_needed(zone1, zone2, zone3, spacing)

# Show a whole number when the result has no leftover.
result = int(result) if result.denominator == 1 else result

# Print the answer with words that fit the scenario.
print("Scanning stations the route needs", result)

Run either in a terminal, enter 2.5, 7/3, 13/6, and 0.25 when prompted, and it returns 28.
