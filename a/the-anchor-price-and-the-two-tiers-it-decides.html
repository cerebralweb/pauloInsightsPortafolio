THE ANCHOR PRICE AND THE TWO TIERS IT DECIDES The Anchor Price And The Two Tiers It Decides

In this article ❧ Vendor pricing rarely arrives as one clean number. A reference quote often anchors a fractional discount and a fixed surcharge, and the buyer has to resolve three values from one. This problem shows how a short sequence of arithmetic turns a layered quote into a single comparable unit, and how a small Python program makes the same calculation repeatable for any future review.

A buyer is comparing three API providers. Provider C charges $1.78 per 1,000 tokens. Provider B charges half of that. Provider A adds 23 cents on top of Provider B. The buyer needs both A and B in the same unit before any side by side comparison is fair.

Converting Provider C to cents gives 178. Halving that gives 89 for Provider B. Adding 23 gives 112 for Provider A. The pair is 112 and 89 cents per 1,000 tokens. The mathematics is linear, and the order matters. The unit conversion has to come before the ratio, and the ratio has to come before the offset. Swap the order and the numbers stop meaning what they should.

The Python version below mirrors that same order. It takes the three inputs, normalizes the anchor into cents, applies the fractional ratio, then adds the offset, and prints both resolved prices with vendor labels.

def compute_provider_prices(price_c_dollars, halving_ratio, offset_cents):
    price_c_cents = price_c_dollars * 100
    price_b_cents = price_c_cents * halving_ratio
    price_a_cents = price_b_cents + offset_cents
    return price_a_cents, price_b_cents

print("Compare API provider prices per 1,000 tokens.")
price_c_dollars = float(input("Provider C price in dollars per 1,000 tokens: "))
halving_ratio = float(input("Halving ratio applied to Provider C (e.g., 0.5): "))
offset_cents = float(input("Additive offset in cents added to Provider B: "))

price_a, price_b = compute_provider_prices(price_c_dollars, halving_ratio, offset_cents)

price_a = int(price_a) if price_a.is_integer() else round(price_a, 2)
price_b = int(price_b) if price_b.is_integer() else round(price_b, 2)

print(f"Provider A charges {price_a} cents per 1,000 tokens.")
print(f"Provider B charges {price_b} cents per 1,000 tokens.")

A second version using the decimal module from the standard library is worth keeping nearby. Floating point arithmetic on money values can drift in the second or third decimal place, which matters once the same calculation is run across many vendors and the rounding errors compound. Decimal gives exact base ten arithmetic, which is how the prices were quoted in the first place.

from decimal import Decimal

def compute_provider_prices(price_c_dollars, halving_ratio, offset_cents):
    price_c_cents = Decimal(price_c_dollars) * Decimal("100")
    price_b_cents = price_c_cents * Decimal(halving_ratio)
    price_a_cents = price_b_cents + Decimal(offset_cents)
    return price_a_cents, price_b_cents

print("Compare API provider prices per 1,000 tokens.")
price_c_dollars = input("Provider C price in dollars per 1,000 tokens: ")
halving_ratio = input("Halving ratio applied to Provider C (e.g., 0.5): ")
offset_cents = input("Additive offset in cents added to Provider B: ")

price_a, price_b = compute_provider_prices(price_c_dollars, halving_ratio, offset_cents)

print(f"Provider A charges {price_a.normalize()} cents per 1,000 tokens.")
print(f"Provider B charges {price_b.normalize()} cents per 1,000 tokens.")

Linear relational reasoning of this kind has deep roots in medieval Islamic mathematics. In the early ninth century the Persian scholar al Khwarizmi, working at the House of Wisdom in Baghdad, wrote the book that gave us the word algebra. A large part of that text dealt not with abstract theory but with practical commercial problems such as inheritance shares, currency exchange, and proportional trade calculations, which look strikingly similar in structure to a modern vendor pricing review.

The point of the math is not the half or the surcharge in isolation. It is the order in which the unit conversion, the ratio, and the offset are applied. The price is decided by the anchor and by that sequence, and the program above is just that sequence written down.
