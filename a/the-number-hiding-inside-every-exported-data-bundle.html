THE NUMBER HIDING INSIDE EVERY EXPORTED DATA BUNDLE

In this article ❧ Data teams rarely see the size of a single table on its own. It usually arrives folded into a larger export, mixed with reference tables that have nothing to do with the metric you care about, so the figure you need is whatever remains once the known weight is stripped out. This problem shows how a short chain of arithmetic can model a storage question, and how Python turns that logic into a tool you can reuse on any bundle.

The Problem

A BI analyst exports one dataset bundle that holds 6 fact tables and 2 dimension tables. The full bundle is 3.80 GB. The 2 dimension tables together take 1.4 GB.

How large is each fact table?

  • a) 0.4 GB
  • b) 4 GB
  • c) 0.40 MB
  • d) 2.4 GB
Click to Reveal the Answer

Correct Answer: a) 0.4 GB ✅

Math Solution

First subtract the dimension tables from the bundle to isolate what the fact tables claim, then divide that remainder across the 6 fact tables.

The trap is the unit. Option c keeps the right figure but swaps GB for MB, so the number is correct and the size is wrong.

Step-by-Step Breakdown

The following sequence shows how the final amount is derived:

  1. Name the unknown: let x be the size of one fact table
  2. Write the equation: 6x + 1.4 = 3.80
  3. Isolate the fact tables: 6x = 3.80 − 1.4 = 2.40
  4. Divide across the tables: x = 2.40 ÷ 6 = 0.40
  5. Conclusion: each fact table is 0.4 GB

Historical Background

Writing the unknown quantity as x is a habit only about four centuries old. René Descartes set out the convention in La Géométrie in 1637, reserving the late letters of the alphabet for unknowns and the early letters for values already in hand.

That single choice is what lets a storage question collapse into one readable line. Accountants and merchants had been splitting a known total into shares for centuries, but the modern notation made the move legible at a glance, and finance, engineering, and data work have leaned on it ever since.

Python Version

This script removes the known weight from any bundle and splits the rest across the fact tables, so it answers the same question for any set of numbers.

def fact_table_size(total_gb, dimension_gb, fact_count):
    fact_total = total_gb - dimension_gb
    size = fact_total / fact_count
    return int(size) if size.is_integer() else size

print("How large is each fact table in the bundle?")
total_gb = float(input("Total bundle size in GB: "))
dimension_gb = float(input("Combined size of the dimension tables in GB: "))
fact_count = int(input("Number of fact tables: "))

size = fact_table_size(total_gb, dimension_gb, fact_count)
print(f"Each fact table is {size} GB")

Run it, enter 3.80, 1.4, and 6, and it returns that each fact table is 0.4 GB.
