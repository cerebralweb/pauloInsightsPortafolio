MULTIPLY A PACKING TIME BY ONE AND A HALF FOR THE NEXT ORDER

In this article ❧ Operations teams rarely plan a job from nothing. They size a new run against one they already measured, scaling that known time to fit a larger or repeat task, which sets the schedule and the staffing window before any work starts. This problem shows how multiplying a known duration by a simple factor turns one measured time into a planning number, and how Python can hold that logic as a reusable tool.

The Problem

A warehouse team packed the first order in 13.5 minutes. The second order took 1 and 1/2 times as long as the first.

How many minutes did the second order take?

  • a) 20
  • b) 6.75
  • c) 2
  • d) 20.25
Click to Reveal the Answer

Correct Answer: d) 20.25 ✅

Math Solution

Read one and a half as the full first time plus half of it again. Multiply 13.5 by 1.5, or add 13.5 and 6.75, and either route lands on 20.25 minutes.

The extra 6.75 minutes is just half of the first order, so the second order is the first order and its half combined.

Step-by-Step Breakdown

The following sequence shows how the final amount is derived:

  1. Full first time: 13.5 minutes
  2. Half of the first time: 6.75 minutes
  3. Add the parts: 13.5 + 6.75 = 20.25 minutes
  4. As mixed numbers: 13 and 1/2 plus 6 and 3/4 gives 19 and 5/4
  5. Carry the extra fourths: 19 and 5/4 = 20 and 1/4
  6. Second order time: 20.25 minutes

Historical Background

Scaling a quantity by halves is one of the oldest moves in arithmetic. Egyptian scribes multiplied by doubling and halving and wrote each fraction as a sum of unit parts, a method preserved in the Rhind Mathematical Papyrus that the scribe Ahmes copied around 1650 BCE. The same instinct, take a known amount and add half of it again, still sits behind everyday estimates of time, cost, and capacity.

Python Version

This script multiplies any first duration by any factor, so it answers the whole family of these problems, not just one set of numbers.

# What this does — works out how long the second order takes when it runs longer than the first.
# Where it came from — a warehouse packed the first order, and the second order took a set number of times as long.
# The math idea — multiply a time by a factor.
# You type in — the first order time in minutes and how many times as long the second order is.
# It prints — the second order time in minutes.

# This function does the math.
def second_order_time(first_minutes, factor):
    # Multiply the first time by the factor.
    return first_minutes * factor

# Tell the reader what we will figure out.
print("Let's find how long the second order takes.")

# Ask for the first order time in minutes.
first_minutes = float(input("How many minutes did the first order take? "))

# Ask how many times as long the second order is. For one and a half, type 1.5
factor = float(input("How many times as long is the second order? "))

# Do the math.
result = second_order_time(first_minutes, factor)

# Show a whole number when it comes out even, else keep the decimal.
result = int(result) if result.is_integer() else result

# Print the answer with words that fit the scenario.
print("The second order takes", result, "minutes.")

The exact version keeps the value as a fraction and also prints the mixed number, which matches the fraction path above.

# What this does — works out the second order time and keeps it exact as a fraction.
# Where it came from — a warehouse packed the first order, and the second order took a set number of times as long.
# The math idea — multiply a time by a factor, shown as a fraction.
# You type in — the first order time and how many times as long the second order is.
# It prints — the answer as a fraction, a mixed number, and a decimal.

# This tool gives exact fractions.
from fractions import Fraction

# This function does the math.
def second_order_time(first_amount, factor):
    # Multiply the first time by the factor.
    return first_amount * factor

# Tell the reader what we will figure out.
print("Let's find the second order time and keep it exact.")

# Ask for the first order time. Type 13.5 or a fraction like 27/2.
first_amount = Fraction(input("How many minutes did the first order take? "))

# Ask how many times as long the second order is. Type 1.5 or 3/2.
factor = Fraction(input("How many times as long is the second order? "))

# Do the math. The answer stays an exact fraction.
total = second_order_time(first_amount, factor)

# Split the fraction into a whole part and a leftover part.
whole = total.numerator // total.denominator
leftover = total - whole

# Build a friendly mixed number text.
if leftover == 0:
    mixed = str(whole)
else:
    mixed = str(whole) + " and " + str(leftover)

# Show the answer three ways so both paths line up.
print("As a fraction, that is", total, "minutes.")
print("As a mixed number, that is", mixed, "minutes.")
print("As a decimal, that is", float(total), "minutes.")

Run either one in a terminal, type the first time and the factor, and it prints the second duration.
