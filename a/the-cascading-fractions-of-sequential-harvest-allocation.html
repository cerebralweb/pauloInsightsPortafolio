THE CASCADING FRACTIONS OF SEQUENTIAL HARVEST ALLOCATION

In this article ❧ Agricultural and supply chain operations depend on sequential allocation logic. Before any quantity reaches its final destination — a market, a warehouse, or a spot channel — it passes through multiple deduction steps, each applied not to the original total but to what remains after the previous cut. This problem demonstrates how cascading fractional deductions work, and how Python can encode that logic into a reusable calculator that adapts to any quantity and any pair of fractions.

The Problem

A farm finished its coffee harvest.

5/9 of the crop was stored in a large silo, 3/4 of the remainder in a small silo, and the rest was sent to the local market. If the total harvest weighed 18 tons, how many tons were sent to the market?

  • a) 2
  • b) 1/9
  • c) 9
  • d) 1
Click to Reveal the Answer

Correct Answer: a) 2 tons ✅

Math Structure

This problem applies sequential fractional reasoning: the second fraction (3/4) is not applied to the original 18-ton total but to the remainder left after the first allocation — a critical distinction that rules out simply adding or combining the two fractions directly.

The key insight is that each deduction changes the base: the first fraction operates on the whole, the second on what survives the first cut.

Step-by-Step Breakdown

The following sequence shows how the final amount is derived:

  1. Total harvest: 18 tons
  2. Large silo (5/9 of total): 5/9 × 18 = 10 tons
  3. Remainder after large silo: 18 − 10 = 8 tons
  4. Small silo (3/4 of remainder): 3/4 × 8 = 6 tons
  5. Sent to market (1/4 of remainder): 8 − 6 = 2 tons

Python Version

The script below generalizes the allocation logic into a reusable function: supply any total, any primary fraction, and any secondary fraction, and it returns the residual quantity that reaches the final destination.

def harvest_to_market(total, large_num, large_den, small_num, small_den):
    large_silo = total * large_num / large_den
    remainder = total - large_silo
    small_silo = remainder * small_num / small_den
    to_market = remainder - small_silo
    result = int(to_market) if to_market == int(to_market) else to_market
    return result

print("--- Coffee Harvest Allocation ---")
total     = float(input("Total harvest weight (tons): "))
large_num = float(input("Large silo fraction — numerator: "))
large_den = float(input("Large silo fraction — denominator: "))
small_num = float(input("Small silo fraction of remainder — numerator: "))
small_den = float(input("Small silo fraction of remainder — denominator: "))

sent = harvest_to_market(total, large_num, large_den, small_num, small_den)
print(f"\nTons sent to the local market: {sent}")

Run it in Windows Terminal by typing python, pasting the block, pressing Enter twice, and supplying each value when prompted. For the original problem the inputs are 18, 5, 9, 3, and 4 — and the program returns Tons sent to the local market: 2.

A second variant using fractions.Fraction from the Python standard library eliminates floating-point rounding errors. It is preferable when the inputs are large or the fractions do not divide evenly into whole numbers.

from fractions import Fraction

def harvest_to_market(total, large_num, large_den, small_num, small_den):
    total      = Fraction(total)
    large_silo = total * Fraction(large_num, large_den)
    remainder  = total - large_silo
    small_silo = remainder * Fraction(small_num, small_den)
    to_market  = remainder - small_silo
    result = int(to_market) if to_market.denominator == 1 else float(to_market)
    return result

print("--- Coffee Harvest Allocation (exact fractions) ---")
total     = int(input("Total harvest weight (tons): "))
large_num = int(input("Large silo fraction — numerator: "))
large_den = int(input("Large silo fraction — denominator: "))
small_num = int(input("Small silo fraction of remainder — numerator: "))
small_den = int(input("Small silo fraction of remainder — denominator: "))

sent = harvest_to_market(total, large_num, large_den, small_num, small_den)
print(f"\nTons sent to the local market: {sent}")

# Harvest Allocation Calculator

This skill takes a total harvest weight and two fractions — one for a primary allocation and one for a secondary allocation applied to the leftover — and calculates exactly how much reaches the final destination (the local market, the spot sale channel, or any residual use). It applies sequential fractional reasoning: compute the first share, subtract it from the total to get the remainder, compute the second share from that remainder, subtract again, and report what is left.

---

## Dependencies

No external libraries required. All arithmetic uses basic Python operators. If exact fractional arithmetic is needed (no rounding), use the `fractions.Fraction` class from the Python standard library.

---

## Inputs

- **Total quantity**: The full harvest weight or stockpile size, expressed as a whole number (e.g., 18 tons). Accept as an integer when the problem guarantees a whole number; accept as a decimal when it may not be.
- **Primary allocation fraction**: The share of the total sent to the first destination, expressed as a numerator and a denominator (e.g., 5 and 9 for five-ninths). Both values must be positive integers.
- **Secondary allocation fraction**: The share of the *remainder* sent to the second destination, also expressed as a numerator and a denominator (e.g., 3 and 4 for three-quarters). Both values must be positive integers.

---

## Steps

### Step 1 — Greet and frame the problem

Print a short header that names the scenario — for example, "Coffee Harvest Allocation" — so the user knows which calculation is about to run.

### Step 2 — Collect the total quantity

Prompt the user for the total harvest weight in tons. Accept a decimal value in case the total is not a whole number.

### Step 3 — Collect the primary allocation fraction

Prompt the user for two separate values: the numerator and the denominator of the fraction that goes to the large silo (or primary destination). Accept both as integers.

### Step 4 — Collect the secondary allocation fraction

Prompt the user for two separate values: the numerator and the denominator of the fraction of the remainder that goes to the small silo (or secondary destination). Accept both as integers.

### Step 5 — Calculate the primary allocation

Multiply the total quantity by the primary fraction (numerator divided by denominator) to find how much goes to the large silo. This is the first deduction from the total.

### Step 6 — Calculate the remainder

Subtract the primary allocation from the total. This intermediate value is the quantity available for the second allocation and the final destination.

### Step 7 — Calculate the secondary allocation

Multiply the remainder by the secondary fraction (numerator divided by denominator) to find how much goes to the small silo. This is the second deduction.

### Step 8 — Calculate the residual sent to market

Subtract the secondary allocation from the remainder. This is the final answer: the quantity that reaches the local market or residual channel.

### Step 9 — Clean and display the result

If the result is a whole number (its decimal part equals zero), display it as an integer rather than a float. Print the result with a clear, friendly label — for example: "Tons sent to the local market: 2".
