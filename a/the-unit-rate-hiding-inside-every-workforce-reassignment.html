THE UNIT RATE HIDING INSIDE EVERY WORKFORCE REASSIGNMENT

In this article ❧ Operations teams constantly shift staff between tasks with different durations — a move that looks simple on a schedule but demands a unit-rate conversion before any realistic capacity figure is possible. Without that conversion, planners risk committing to output numbers that are off by a factor of two or more. This problem illustrates how deriving a per-task rate from a known baseline unlocks precise shift-capacity planning, and how Python can package that logic into a reusable tool any analyst can run on demand.

The Problem

A solutions consultant at a SaaS company normally spends 30 minutes per client onboarding call. Today, however, her manager has asked her to run software license audit checks instead. She can complete 2 license audits in the same time it takes to conduct one onboarding call. If she has 3.5 hours remaining in her workday, how many license audits will she be able to complete?

  • a) 30
  • b) 28
  • c) 7
  • d) 14
Click to Reveal the Answer

Correct Answer: d) 14 ✅

The key insight is that the conversion rate runs in one direction only — 2 audits fit inside 1 onboarding slot, so each audit takes 30 ÷ 2 = 15 minutes, not 30 × 2 = 60 minutes.

Step-by-Step Breakdown

The following sequence shows how the final count is derived:

  1. Duration of one onboarding call: 30 minutes (given)
  2. Audits per onboarding slot: 2 audits fit inside 30 min → each audit = 30 ÷ 2 = 15 minutes
  3. Convert shift time to minutes: 3.5 hours × 60 = 210 minutes
  4. Number of audits: 210 ÷ 15 = 14
  5. Conclusion: With 3.5 hours remaining, the consultant can complete 14 license audits — answer d)

Why This Math Matters for the Business Problem

In SaaS operations and consulting, staff time is a constrained resource that must be allocated across tasks with different durations. When a manager reassigns a consultant from one task type to another mid-day, the team needs to know quickly how many units of the new task fit into the remaining capacity — without guessing. Unit rate conversion gives an exact answer in seconds, enabling realistic workload planning, SLA commitments, and shift scheduling.

In this scenario, knowing that each license audit takes 15 minutes (not 30) is what unlocks the correct capacity figure. Getting it wrong by using the onboarding rate instead would produce an estimate of 7 — cutting the planned output in half and potentially leaving compliance work unfinished.

Mathematical Background

This problem applies unit rate conversion, a foundational technique in ratio and proportion arithmetic. The core idea is to express a quantity in terms of a single unit — here, minutes per audit — so that a straightforward division yields the total count. The process has two moves: derive the new rate from a known relationship (2 audits per 30 min → 15 min/audit), then divide total available time by that rate.

Unit rates underpin nearly every capacity and throughput calculation in operations management. A useful conceptual fact: the unit rate is simply a ratio reduced so that the denominator equals 1 — a normalization technique that appears across disciplines, from physics (meters per second) to finance (cost per unit) to HR (calls per agent per hour).

Python Version

The script below generalizes the problem: it accepts any baseline task duration, any conversion rate, and any remaining shift length, then returns the maximum number of complete tasks — using math.floor() to exclude partial work, which is the operationally honest answer.

Option 1 — Pure Python

def license_audits_in_shift(baseline_minutes, audits_per_slot, shift_hours):
    audit_duration = baseline_minutes / audits_per_slot
    shift_minutes = shift_hours * 60
    return shift_minutes / audit_duration

print("License Audit Capacity Planner")
print("How many audits can a consultant complete in the remaining shift?\n")

baseline_minutes = float(input("Duration of one onboarding call (minutes): "))
audits_per_slot  = float(input("How many audits fit inside one onboarding slot? "))
shift_hours      = float(input("Remaining shift time (hours): "))

result = license_audits_in_shift(baseline_minutes, audits_per_slot, shift_hours)
result = int(result) if result.is_integer() else result

print(f"\nLicense audits completable in remaining shift: {result}")

Option 2 — Standard Library (math.floor)

math.floor() ensures the result reflects only fully completable audits — a consultant cannot deliver half an audit, and a capacity plan that rounds up overstates output. Run either option in Windows Terminal by typing python, pasting the code, and entering the three prompted values.

import math

def license_audits_in_shift(baseline_minutes, audits_per_slot, shift_hours):
    audit_duration = baseline_minutes / audits_per_slot
    shift_minutes  = shift_hours * 60
    return math.floor(shift_minutes / audit_duration)

print("License Audit Capacity Planner")
print("How many complete audits can a consultant finish in the remaining shift?\n")

baseline_minutes = float(input("Duration of one onboarding call (minutes): "))
audits_per_slot  = float(input("How many audits fit inside one onboarding slot? "))
shift_hours      = float(input("Remaining shift time (hours): "))

result = license_audits_in_shift(baseline_minutes, audits_per_slot, shift_hours)

print(f"\nLicense audits completable in remaining shift: {result}")

Reusable Skill — XML

The skill document below packages the capacity-planning logic as a structured Claude skill. It defines inputs, calculation steps, and a quality checklist that generalizes the script to any task-reassignment scenario.

name: shift-capacity-calculator
description: >
  Calculates how many units of a reassigned task a worker can complete in
  remaining shift time, given the baseline task duration and the conversion
  rate between the two task types. Trigger this skill whenever the user wants
  to know how many tasks, audits, checks, or service items fit into a time
  window after a workload reassignment.
---

    maximum number of complete units of the new task that fit inside the

    completable tasks, not fractions of work.
  </purpose>

  </dependencies>

    </input>

      <name>audits_per_slot</name>
      <description>
        How many units of the new task fit inside one baseline task slot.

      </description>
    </input>
    <input>
      <name>shift_hours</name>
      <description>
        Remaining shift time in hours. Must be a positive number. Accept

      </description>

  <steps>

    <step id="1" name="Collect Inputs">
      Print a short title and one-line description of the tool's purpose.

      remaining shift time in hours. Cast each value to float.
    </step>

      Divide the baseline task duration by the number of new tasks per slot.

    </step>

    <step id="3" name="Convert Shift Time to Minutes">
      Multiply the remaining shift hours by 60 to express the available window

      Print the floored integer result with a short, friendly label that

      Example output: "License audits completable in remaining shift: 14"
    </step>

  </steps>
