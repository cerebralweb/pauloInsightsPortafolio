THE INVERSE OPERATION THAT RECOVERS WHAT THE NARRATIVE BURIES

In this article ❧ When companies audit historical contracts, they often find only the current figure and the factor by which it grew. That single ratio, applied in reverse, is enough to reconstruct what was paid years earlier, but only if the equation is stated before any arithmetic begins. This article works through a standard pricing problem using inverse operation, shows the Python program that automates the recovery, and packages the logic as a reusable Claude skill.

The Problem

In 2024, a software license for a workflow automation platform cost $3.28 million for one client. That price was eight times what the same license would have cost in 2014.

How much did the license cost in 2014?

  • a) $26.24 million
  • b) $41 million
  • c) $4.1 million
  • d) $0.41 million
Click to Reveal the Answer

Correct Answer. d) $0.41 million ✅

Math Solution

The 2024 price is eight times the 2014 price, so the 2014 price is the 2024 price divided by eight. Dividing $3.28 million by 8 gives $0.41 million — option d.

Multiplication and division are inverse operations. If you know the product and one factor, division always recovers the missing value.

Step by Step Breakdown

The following sequence shows how the final amount is derived.

  1. Define the unknown. Let x = the 2014 license price in millions of dollars.
  2. Write the equation. Since the 2024 price equals 8 times the 2014 price, write 8x = 3.28.
  3. Isolate x. Divide both sides by 8 → x = 3.28 ÷ 8.
  4. Compute. 3.28 ÷ 8 = 0.41 → x = 0.41.
  5. Conclusion. The 2014 license cost $0.41 million, which matches option d.

Historical Background

The systematic treatment of equations with one unknown was laid out in the 9th century by the Persian mathematician al-Khwarizmi, writing in Baghdad. The word algebra comes from an Arabic term in his title referring to the act of restoring and balancing equations. The method he described, isolating the unknown by applying an inverse operation to both sides, is exactly what this problem requires.

Python Version

The program reads any narrative containing a dollar amount and a multiplier in digit form ("8 times") or word form ("eight times"), states the raw equation, and on confirmation returns the solvable form and the answer.

import re

# --- HELPERS ---

# Maps spelled-out numbers to digits
WORD_TO_NUM = {
    "one": 1, "two": 2, "three": 3, "four": 4, "five": 5,
    "six": 6, "seven": 7, "eight": 8, "nine": 9, "ten": 10
}

def extract_from_narrative(text):
    # Reads the text, grabs the dollar amount and the multiplier, ignores everything else
    dollar_match = re.search(r'\$\s*([\d,]+\.?\d*)', text)
    known_value = float(dollar_match.group(1).replace(',', '')) if dollar_match else None

    # Tries digit first ("8 times"), then spelled-out word ("eight times")
    mult_digit = re.search(r'(\d+)\s*times', text, re.IGNORECASE)
    mult_word  = re.search(r'(\w+)\s+times', text, re.IGNORECASE)

    if mult_digit:
        multiplier = int(mult_digit.group(1))
    elif mult_word and mult_word.group(1).lower() in WORD_TO_NUM:
        multiplier = WORD_TO_NUM[mult_word.group(1).lower()]
    else:
        multiplier = None

    return known_value, multiplier

def raw_equation(known, mult):
    # Builds the unresolved equation: known = mult * x
    return f"{known} = {mult}x"

def solvable_equation(known, mult):
    # Flips the equation so x is alone: x = known * (1/mult)
    return f"x = {known} * (1/{mult})"

def solve(known, mult):
    return round(known * (1 / mult), 2)

# --- STAGE 1: extract and show raw equation ---

print("Paste the narrative statement.")
s = input("> ")

known, mult = extract_from_narrative(s)

if known is None or mult is None:
    print("No dollar amount or multiplier found. Stopping.")
else:
    print(f"\nClutter removed. Raw equation\n   {raw_equation(known, mult)}")

    # --- STAGE 2: confirm, then solve ---

    print("\nPaste the same narrative to transform the equation.")
    s2 = input("> ")

    k2, m2 = extract_from_narrative(s2)

    if k2 == known and m2 == mult:
        print(f"\nSolvable equation\n   {solvable_equation(known, mult)}")
        print(f"\nEarlier price recovered. x = ${solve(known, mult)}")
    else:
        print("Second statement does not match. No transformation performed.")

Run in Windows Terminal, paste the narrative at the first prompt, read the raw equation, paste the same narrative again, and read the recovered earlier price.

Claude Skills Version

The skill file below translates the program into plain English steps. Claude follows them without executing code. The structure mirrors the program: extract, state, gate, transform, solve.

---
name: narrative-equation-solver
description: Reads a plain English statement that describes one value as a multiple of an
unknown earlier value. Strips clutter, states the raw equation, and on confirmation transforms
it into solvable form to recover the unknown. Trigger whenever the user pastes a narrative
price or quantity relationship and wants the unknown value recovered.
---

  <purpose>

    State the raw equation.

    </input>
    <input name="narrative_second">
      The same narrative pasted a second time. Acts as a confirmation gate before the transformation runs.
    </input>
  </inputs>

  <steps>
    <step id="1" name="Build the word to number map">
      Before reading any input, prepare a lookup table that maps spelled-out numbers one through
      ten to their digit equivalents. This handles multipliers that appear as words rather than digits.
    </step>

    <step id="2" name="Read the first narrative">
      Prompt the user to paste the narrative statement. Accept free-form text with no required format.
    </step>

    <step id="3" name="Extract the dollar amount">
      Scan the text for a dollar sign followed by a numeric value (integer or decimal, with optional
      commas). Store it as the known value. If not found, stop and report that no dollar amount was detected.
    </step>

    <step id="4" name="Extract the multiplier">
      Scan for a number followed by the word "times". Try digit form first. If not found, try word

      stop and report the mismatch without transforming.
    </step>

    <step id="7" name="Transform and solve">
      Rearrange the equation so x is isolated.
      Format: x = known_value times (1 divided by multiplier)
      Compute the result. Round to two decimal places. Display the solvable equation and the recovered
      earlier value with a short label.
    </step>
  </steps>

  <conditional_logic>
    <condition>If the dollar amount or multiplier is missing after the first input, stop and report which value is absent.</condition>
    <condition>If the second narrative does not match the first, stop without transforming and report the mismatch.</condition>

  </conditional_logic>

    <output stage="Stage 1">Raw equation printed to the terminal: known_value = multiplier x</output>

    followed by the computed earlier value labeled as the recovered unknown.</output>
  </outputs>
