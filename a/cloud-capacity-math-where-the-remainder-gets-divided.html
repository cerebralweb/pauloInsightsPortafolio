CLOUD CAPACITY MATH WHERE THE REMAINDER GETS DIVIDED

In this article ❧ Capacity planning rarely starts with an empty pool. Some portion of every resource is already committed or reserved before equitable allocation begins, whether the resource is storage, budget, headcount, or bandwidth. This problem shows how a short arithmetic procedure can model that reality, and how a small Python program can turn the logic into a reusable tool for any team facing the same decision.

The Problem

A DevOps team has 128 GB of storage to provision across 12 virtual machines for a new project. They configure 5 machines with 15.75 GB each. From the remaining capacity, they split storage equally across the other 7 machines. The team also reserves 2 GB for system overhead that cannot be allocated.

How much storage does each of the 7 machines receive?

  • a) 4
  • b) 6.968
  • c) 6.75
  • d) 11
Click to Reveal the Answer

Correct Answer: c) 6.75 GB ✅

Math Solution

The procedure is a two stage arithmetic. First, subtract the storage already committed to the 5 fixed machines and the reserved overhead from the total capacity. Then, divide the remaining capacity into equal shares for the 7 unassigned machines.

Every allocation problem hides the same conversion. Total minus committed minus reserved gives the pool that division operates on.

Step-by-Step Breakdown

The following sequence shows how the final amount is derived:

  1. Storage for the 5 fixed machines: 5 × 15.75 = 78.75 GB committed to known workloads
  2. Total committed plus reserve: 78.75 + 2 = 80.75 GB unavailable for the remaining machines
  3. Remaining capacity: 128 − 80.75 = 47.25 GB pool available for equal split
  4. Per machine allocation: 47.25 ÷ 7 = 6.75 GB equal share for each of the 7 machines

Historical Background

The combination of subtraction and division to allocate a remainder is one of the oldest practical computations in commerce. The Treviso Arithmetic, printed in Venice in 1478, was the first arithmetic textbook produced on a printing press, and it was written for merchants and apprentices who needed to handle exactly this kind of calculation. Its problems involved splitting a shipment after costs, distributing profits among partners after deducting expenses, and dividing leftover goods among buyers. The book helped establish arithmetic as a working tool for traders and clerks rather than a purely scholarly subject, and the same logic now governs how teams allocate cloud storage, software licenses, and operating budgets.

Python Version

The script accepts any total capacity, any number of fixed machines with any preset size, any reserved overhead, and any count of remaining machines, then returns the equal share for each unassigned machine. It is reusable for storage, budget, license, and bandwidth allocation problems with the same structure.

def storage_per_remaining_machine(total_capacity, fixed_count, fixed_size, reserved, remaining_count):
    committed = fixed_count * fixed_size
    available = total_capacity - committed - reserved
    per_machine = available / remaining_count
    return int(per_machine) if per_machine.is_integer() else per_machine

print("Cloud storage allocation across virtual machines")
print("How much storage does each remaining VM receive after fixed allocations and reserved overhead?")
print()

total_capacity = float(input("Enter total storage capacity in GB: "))
fixed_count = int(input("Enter the number of machines with fixed allocations: "))
fixed_size = float(input("Enter the storage per fixed machine in GB: "))
reserved = float(input("Enter the reserved overhead in GB: "))
remaining_count = int(input("Enter the number of remaining machines that share leftover capacity: "))

result = storage_per_remaining_machine(total_capacity, fixed_count, fixed_size, reserved, remaining_count)

print()
print(f"Each of the {remaining_count} remaining virtual machines receives {result} GB of storage.")

Run it in a terminal by opening Python, pasting the block, and entering the five values when prompted. The final line returns the per machine allocation in plain language.
