THE NUMBER HIDING INSIDE A DATA CENTER POWER BILL

In this article ❧ Infrastructure costs arrive as bulk totals that hide what each machine consumes. Before a team can budget, forecast, or size cooling, it must turn one figure into a rate per machine per period. This problem shows how a chain of division converts a bill into a planning unit, and how Python turns that logic into a tool.

The Problem

A data center runs 20 servers. Over 4 weeks, those servers consume 9,732.8 kWh of electricity. Assuming each server consumes the same amount, the team needs the consumption for one server.

How much electricity does one server consume in a week?

  • a) 86.66
  • b) 120
  • c) 87
  • d) 122
Click to Reveal the Answer

Correct Answer: d) 122 ✅

Math Solution

Multiply 20 servers by 4 weeks to get 80 server weeks. Divide 9,732.8 by 80 to get 121.66, which rounds to 122.

Collapsing two divisions into one division by 80 turns the whole problem into a single step.

Step-by-Step Breakdown

The following sequence shows how the final amount is derived:

  1. Total billing units: 20 servers × 4 weeks = 80 server weeks
  2. Rate per unit: 9,732.8 ÷ 80 = 121.66 kWh
  3. Round to whole: 121.66 becomes 122
  4. Conclusion: one server consumes about 122 kWh per week

Historical Background

Merchants have reduced bulk quantities to per unit rates since antiquity, when traders split a shipment across partners and seasons to price a single share. Babylonian scribes kept reciprocal tables so they could turn division into multiplication and settle accounts fast. The same move underlies cost allocation today, where one invoice becomes a rate per server, per seat, or per hour.

Python Version

This script reduces any bulk total to a rate per resource per period, so it works for power bills, fuel, or fleet usage alike.

# What this does: works out how much electricity one server uses in one week.
# Where it came from: a data center ran some servers for some weeks and got one big power total.
# The math idea: divide the big total down to one server for one week.
# You type in: the total power used, the number of servers, and the number of weeks.
# It prints: the power one server uses in one week.

# This function does the math.
def power_per_server_per_week(total_kwh, servers, weeks):
    # Count how many server weeks the total covers.
    server_weeks = servers * weeks
    # Split the big total evenly across all those server weeks.
    return total_kwh / server_weeks

# Tell the reader what we will figure out.
print("Let's find how much power one server uses in one week.")

# Ask for the total power used.
total_kwh = float(input("Total electricity used in kWh: "))

# Ask for the number of servers.
servers = int(input("Number of servers: "))

# Ask for the number of weeks.
weeks = int(input("Number of weeks: "))

# Do the math.
result = power_per_server_per_week(total_kwh, servers, weeks)

# Round to two decimals so it reads cleanly.
result = round(result, 2)

# Show a whole number when there is no leftover.
result = int(result) if result == int(result) else result

# Print the answer with words that fit the scenario.
print("Electricity one server uses per week in kWh:", result)

Run it in a terminal, enter 9732.8, then 20, then 4, and it returns 121.66 kWh per server per week.

# Unit Rate Per Resource Calculator

This skill takes a bulk total spread across a number of resources and a number of time periods, then works out how much a single resource accounts for in a single period. The input is three numbers: the total amount, the count of resources, and the count of periods. The output is one number stated in plain business terms, the amount per resource per period.

---

## Dependencies

None. This uses basic arithmetic only. No libraries are needed.

---

## Inputs

- **Total amount**: The full bulk figure to split, for example total electricity in kWh. May be a decimal.
- **Number of resources**: How many resources shared the total, for example servers. A whole number.
- **Number of periods**: How many time periods the total covers, for example weeks. A whole number.

---

## Steps

### Step 1 — Greet and state the goal

Print a short line telling the reader what will be figured out, framed to the scenario, for example finding how much power one server uses in one week.

### Step 2 — Collect the three inputs

Ask for the total amount and accept a decimal. Ask for the number of resources and accept a whole number. Ask for the number of periods and accept a whole number.

### Step 3 — Count the resource periods

Multiply the number of resources by the number of periods. This gives the total count of resource periods the bulk figure covers, for example server weeks.

### Step 4 — Divide the total

Divide the total amount by the resource period count from Step 3. The result is the amount for one resource in one period.

### Step 5 — Clean the result

Round the result to two decimals so it reads cleanly. If the rounded result has no leftover decimal, show it as a whole number instead.

### Step 6 — Report the answer

Print the result with short text that fits the scenario, naming the resource and the period, for example electricity one server uses per week.
