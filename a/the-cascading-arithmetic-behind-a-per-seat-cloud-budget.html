THE CASCADING ARITHMETIC BEHIND A PER SEAT CLOUD BUDGET

In this article ❧ Cloud budgets are set per seat and divided across services that scale together. When two services are priced as fractions of a third, their costs stop being independent, and the whole allocation reduces to one base price. This problem shows how a short chain of arithmetic models a real budgeting decision, and how Python turns that logic into a reusable tool.

The Problem

A SaaS company sets a monthly cloud budget of $184 per seat. After provisioning three services for each seat, $39.50 per seat remains unspent. The team provisioned a virtual machine instance, a managed database service that costs 3/5 of the VM instance price per seat, and a monitoring agent license that costs 1/6 of the managed database price per seat.

How much does the virtual machine instance cost per seat?

  • a) $85
  • b) $110
  • c) $59
  • d) $50
Click to Reveal the Answer

Correct Answer: a) $85 ✅

Math Solution

The three services sum to the spent amount, $184 minus $39.50, which is $144.50. The database is 3/5 of the VM and the monitoring is 1/10 of the VM, so the costs collapse into 1.7 times the VM price. Dividing 144.50 by 1.7 returns 85.

Two of the three prices are defined off the first, so one base price drives the entire allocation.

Step-by-Step Breakdown

The following sequence shows how the final amount is derived:

  1. Spent total: 184 minus 39.50 equals 144.50
  2. Database part: 3/5 of the VM, or 0.6 v
  3. Monitoring part: 1/6 of the database, or 0.1 v
  4. Combined multiple: 1 plus 0.6 plus 0.1 equals 1.7
  5. VM price: 144.50 divided by 1.7 equals 85
  6. Check: 85 plus 51 plus 8.50 plus 39.50 equals 184

Bar Models

The budget per seat splits into the spent amount and the leftover.

The three services as fractions of the VM price. The VM is the full bar, the database six tenths of it, the monitoring a sliver.

The unit method. Slice v into ten units. The three bars stack to seventeen units equal to 144.50, so one unit is 8.50 and the VM is ten units, 85.

Historical Background

Recovering an unknown quantity from a fraction of itself is one of the oldest recorded calculations. The Rhind Mathematical Papyrus, copied in Egypt around 1550 BCE, holds aha problems in which a quantity and a part of itself sum to a stated value. The same backing out of a base figure from a known total now sizes budgets, contracts, and tiered pricing across business.

Python Version

The script reads a budget, a leftover, and two fractions, then returns the base price, so it serves any three item split with the same structure.

# What this does: finds the VM price per seat when two other service costs are fractions of it.
# Where it came from: a SaaS budget per seat was split across three cloud services, with some left over.
# The math idea: add up the fractions into one multiple, then divide the spent total by it.
# You type in: the budget, the leftover, and the two fractions.
# It prints: the VM instance price per seat.

# This function does the math.
def vm_price(budget, leftover, db_top, db_bottom, mon_top, mon_bottom):
    # Money actually spent on the three services.
    spent = budget - leftover
    # Database cost as a part of the VM price.
    db_part = db_top / db_bottom
    # Monitoring is a part of the database, so a part of a part of the VM price.
    mon_part = (mon_top / mon_bottom) * db_part
    # Add the three parts. The VM itself is one whole.
    total_parts = 1 + db_part + mon_part
    # Divide the spent total by the parts to get the VM price.
    return spent / total_parts

# Tell the reader what we will figure out.
print("Let's find the VM instance price per seat.")

# Ask for the budget per seat.
budget = float(input("Budget per seat: "))

# Ask for the leftover per seat.
leftover = float(input("Leftover per seat: "))

# Ask for the database fraction of the VM (top and bottom).
db_top = float(input("Database fraction top (e.g. 3): "))
db_bottom = float(input("Database fraction bottom (e.g. 5): "))

# Ask for the monitoring fraction of the database (top and bottom).
mon_top = float(input("Monitoring fraction top (e.g. 1): "))
mon_bottom = float(input("Monitoring fraction bottom (e.g. 6): "))

# Do the math.
result = vm_price(budget, leftover, db_top, db_bottom, mon_top, mon_bottom)

# Round money to two decimals.
result = round(result, 2)

# Show a whole number when it lands on one.
result = int(result) if float(result).is_integer() else result

# Print the answer with words that fit the scenario.
print("VM instance price per seat:", result)

Run it, enter 184, 39.50, 3, 5, 1, 6, and it returns 85.

# Cloud Budget Base Price Solver

Finds the base price per seat when a budget is split across three services, two of them priced as fractions of the base, with some money left over.

## Inputs

- Budget per seat
- Leftover per seat
- Database fraction of the base, a top and a bottom
- Monitoring fraction of the database, a top and a bottom

## Steps

### Step 1 — Find the spent total

Subtract the leftover from the budget. This is the money that went to the three services.

### Step 2 — Express each cost as a part of the base

The base counts as one whole. The database is its fraction of the base. The monitoring is a fraction of the database, so a fraction of a fraction of the base. Reduce monitoring to a single part of the base.

### Step 3 — Add the parts

Add one whole plus the database part plus the monitoring part. This gives one multiple of the base price.

### Step 4 — Solve for the base

Divide the spent total by that multiple. The result is the base price.

### Step 5 — Check

Rebuild the three costs from the base. Confirm they sum to the spent total and that budget minus spent equals the leftover.

State the result naming the service, e.g. VM instance price per seat 85, as a whole number when it lands on one.
